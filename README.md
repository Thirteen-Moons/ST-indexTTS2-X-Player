# ST-IndexTTS2-X-player🎙️

  
为SillyTavern设计的文本转语音（TTS）扩展。

基于原始插件 [indexTTS player](https://github.com/bronie-honkai/st-indextts2) 的二次修改[st-indextts2-plus](https://github.com/xiaoxiongweihu/st-indextts2-plus)，再次进行开发与优化，让你的SillyTavern语音体验更加流畅。

版本：1.0.0 | 作者：Thirteen-Moons

---
## 💡适用人群

  - 想使用本地语音模型，且电脑显存达8G。

  - 此插件不限于PC或手机端SillyTavern，仅是语音模型后端需在电脑上运行，前端面板仅填入API即可使用。

---

## 🎁 功能特色

### 🎤 多角色智能配音

  - 为每个角色单独配置音频，自动匹配朗读
  - 快速管理与切换角色配音


### 📚 灵活的文本解析模式

  - 听书模式：全文朗读，但会过滤掉所有不应被朗读出的符号
  - RP模式：仅朗读引号内对话
  - GAL模式：支持单卡多角色分别配音、场景音效，仅朗读带格式标签的台词

  
### ⚡实时自动流式播放

  - 智能分段：智能切割文本，让推理无需等待
  - 自动播放：收到回复后自动生成语音并播放

  
### 💓 情感向量

  - 支持8个维度的情感向量，生成更富有情感的语音

  
### 🖱️ 悬浮播放器

  - 播放/暂停控制
  - 进度条拖拽
  - 音量与速度控制
  - 可选隐藏

  
### 🏠 场景配音

  - 为不同场景分别配置背景音
  - 自动播放
  - 可选循环播放

  
### 💻 多预设管理

   - 支持保存与切换多套配音方案

  
### 💾 音频缓存

  - 本地音频缓存，已生成的语音无需重复推理即可随时播放
  - 支持导入/导出音频缓存
  - 管理：自定义缓存路径、清空缓存

## 🔧 功能新增与修复

### 🔨 bug修复

  - 音色绑定持久化：修复音色掉绑定的偶发bug。
  - 修复音量越界的偶发报错。
  - 修复悬浮播放器在移动端不显示的bug。
  - 大幅简化代码，运行与推理更流畅。
  
### ⭐ 新增功能

  - **正则过滤**：
    * 未勾选自定义正则过滤时，采用代码内的硬过滤，即便是复杂的MD、json等文本，均能过滤干净（少量emoji除外），仅朗读正常的对话部分。
    * 可选自定义正则：勾选启用正则过滤时，仅使用填入的正则。

  
  - **实时伪流式分段**：新增分段逻辑，将需要朗读的长文本过滤后再进行切割，每推理完一句立即播放，并同时推理下一句，达到更佳的实时输出的效果。


  - **RP解析模式**：适用于常见小说体RP，仅读引号内，支持中、英、直角引号。
    * 可选功能：延迟播放。可根据阅读速度设置延迟。  


  - **真群聊配音**：在st的真群聊中（即多个角色卡组合成一群聊），无需启用提示词注入，即可为每个角色绑定不同的音色。三种解析模式均支持。

  
  - **单人模式**：在单角色的情况下进行音色绑定，无需额外启用提示词注入。三种解析模式均支持。


  - **GAL模式格式兼容**：格式兼容性增强。现对话内容的包裹符号，支持中、英、直角三种引号，与无引号。

  
  - **隐藏播放器**：可选隐藏悬浮播放器。

## 📖 使用说明

### 📦 安装指南

1. 将本仓库克隆或下载到 SillyTavern 的扩展目录：

   ```
   https://github.com/Thirteen-Moons/st-indextts2
   ```

2. 下载Index-tts-2.0 整合包：
   ```
   https://tcnlo9s668u9.feishu.cn/wiki/KRQ9wuqiViSOmJkfHxacNFfknjh
   ```

  
3. 启动 IndexTTS2 后端语音模型服务，路径project/IndexTTS/nvidia/api.bat
    * 注意：推理与输出速度取决于个人显卡性能，显卡越强，速度越快。

4. 在 SillyTavern 扩展面板中启用 **IndexTTS2 Player**
  ---
### 🚀 快速配置


1. API地址：保持默认即可。若需使用情感向量，则将 `TTS服务地址` 改为 `http://127.0.0.1:7880/api/v1/tts/tasks`
    * 若PC端播放正常，但手机局域网播放报错，则将三个地址的 127.0.0.1:7880 改为本地ipv4地址。


2. 在后端语音模型服务的文件夹中，添加参考音频。
    * 角色参考音频文件放置于后端 `api/ckyp/` 目录（若无则自行新建），下拉框会自动列出所有可用文件。
    * 场景音频文件放置于后端 `api/pjy/` 目录（无则新建），每次重启酒馆都需点击授权才能生效。场景音仅适用于GAL模式，文件名需与标签名一致。
    * 注意：参考音频的长度最好在10-20秒内，包含陈述句、语气词、问句、笑声的音频效果会更佳。实际语音效果与音质，取决于参考音频质量。

  
3. 点击待对话的 AI 消息旁的 ⚙ 按钮，打开配音配置面板。输入角色姓名并保存，下拉选择需要使用的声音，绑定。

  
4. 在插件设置面板中，选择需要的解析模式。听书模式、RP模式、GAL模式选一。  

  
5. 勾选「回复后自动推理」和「推完N句后自动续播」，推理句数推荐设为 1。

  
6. 开始聊天。

## 🎵 文本解析模式说明


### ✅ 听书模式

  全文朗读，会过滤掉所有不应朗读出的符号与内容。如MD符号、代码块、链接、斜体内容、星号、xml标签等。

  **适合听故事、人机对话、常见的括号体RP对话。**


**解析效果**
```
【原文一】：
### 问题诊断：为什么移动端不显示
你的临时测试代码能工作，说明基础显示逻辑（display、opacity、z-index）没问题。问题出在**正式CSS中的特定属性组合在移动端被错误解析**。
  
【实际朗读】：
问题诊断：为什么移动端不显示
你的临时测试代码能工作，说明基础显示逻辑没问题。问题出在正式CSS中的特定属性组合在移动端被错误解析。

  
【原文二】：
（攥紧了拳头，指节因用力而微微发白）我…不是这个意思。（声音因痛苦而颤抖着）
  
【实际朗读】：
我…不是这个意思。


【原文三】：
*攥紧了拳头，指节因用力而微微发白* 我…不是这个意思。*声音因痛苦而颤抖着*

【实际朗读】：
我…不是这个意思。

  
  （注：逗号句号省略号等正常符号均不会被朗读出）
```

---
### ✅RP模式

  只朗读引号内，支持中、英、直角三种引号，**适合常见的小说体RP对话**

  可根据不被朗读出的描写段落的大概长度，设置延迟播放下一句的秒数。

  
**解析效果**

```
【原文】：
夜风掠过废弃的站台，铁轨上长满了枯黄的杂草。阿哲把背包甩到肩上，抬头看向站台尽头模糊的信号灯。
“火车应该还有十分钟进站。”他看了看表，声音压得很低。
站台角落的路人一边皱着眉头翻动手中的旧地图，一边打量着这个年轻人。此时远处传来低沉的汽笛声，划破了雨夜的寂静。
   
【实际朗读】：
火车应该还有十分钟进站。
```

--- 
### ✅GAL模式

  **唯一支持单卡多角色分别配音、情感向量、场景音的模式**，但需开启提示词注入。

  
**解析效果**

```
【原文一】：
[大刘][开心]「哈哈，今天居然中了奖！」
兴奋地挥舞着手中的彩票，笑得眼睛都眯成了一条缝。

[小李][讽刺]「瞧你那样儿，当心乐极生悲。」
他不屑地说道，眼睛却时不时瞟向大刘手中的彩票。

【实际朗读】：
哈哈，今天居然中了奖！
瞧你那样儿，当心乐极生悲。


（注：实际朗读为两个角色各自的声音。由齿轮面板中分别设置两者的声音，并让AI在对话中输出角色名标签，即可实现单卡多角色配音）

  
【原文二】：
[小波][兴奋][夏季公园]“哎呀，你看那边的云长得真像一只猫。”
一个小孩兴奋地指向天空，声音清脆悦耳。
  
【实际朗读】：
哎呀，你看那边的云长得真像一只猫。

  
（注：在文件夹设置了对应场景音后，若输出标签，则会在朗读时带有背景音效。此句应为，小波用兴奋的语气说话，并配有夏季公园的背景音）
```

---  
#### GAL模式支持的消息格式

```
对话内容部分，支持中、英、直角三种引号包裹，与无引号。

前缀标签一至三标签均可。

若需要情感向量与场景音则必须加入 [表情] 与 [场景] 标签。

  
[角色名][表情][场景]「对话内容」

[角色名][表情]「对话内容」

[角色名]「对话内容」  
```

*GAL模式必须开启提示词注入，引导 AI 统一使用上方格式输出。*


**提示词示例**：
```
# 格式输出规范
**描写任何角色（主要角色、NPC、路人、旁白）说话时，必须严格遵守格式，对话单开一行**
  
## 格式：
[角色名][情感][场景]「对话内容」
  
### 角色名：
当前说话的人物名称。
  
### 情感：
每个角色仅选择其中一个适配于当下情境的使用，如：
开心、愤怒、悲伤、恐惧、厌恶、忧郁、惊讶、平静
  
### 对话内容：
用「」或 “” 包裹角色说出的台词。
  
### 场景：
仅从以下两个分类中**选择一个**使用，仅在列表选择，避免自创。NSFW内容出现时，需切换到NSFW场景音，若无对应场景可以不填场景。
#### 正常场景列表：
乡村清晨、公园氛围、厨房_切菜、雨声、城镇闹市、沙滩海鸥鸣叫、平稳的心跳声、浴室_淋浴
#### NSFW场景列表：
床的摇晃声、浴室性爱声、女性轻微呻吟、男性喘息
  
## 格式示例：

[小明][开心][春日公园]「今天的天气真好呢。」
他悠闲地在公园中漫步，看着来往的人群。
  
[小方][惊讶]「啊？？你刚刚说什么？」
震惊地侧过头，似乎不敢相信自己听到了什么。
```

  *注意：场景根据你自己的背景音更改，比如你场景音文件是 公园.mp3，那么提示词里的[场景]就必须是 “公园” 这个词。*
